FINGER BANG

SOLO PROJECT

Unity (3D project)

Game Overview

What is the game?

The game is set in the Unknown where you must shoot a waves of enemies coming towards you, if you do not kill them then they will kill you. Have you fingers ready because this will be a bang

Why create this game?

The project I decided to do was a game project for the leap motion using machine learning in which the player uses their fingers to shoot enemies and using voice commands to activate certain abilities such as freezing the enemies or healing the player. The intension for this is to see how the learning algorithm set for this would work and what algorithms could improve the features and training sets.

Gameplay

You are able to be in control, your fingers are your weapon. Using a leap motion to detect hand motion, you must use your fingers to shoot the enemies as fast as you can and do NOT let the enemies get you first. This is done by a combination of softwares: Unity, Processing & Wekinator. Using a Processing sketch which detects your hand motion, Unity which has the receivers to execute the code and Wekinator which communicates between both input and output.

Hardware & Software

Unity

It was used to create the game in which the player is faced in the certain fixed position and must shoot enemies appearing at random locations. There are scripts for the player, enemy, collision and a day/night cycle, which were written in C# also the OSC and UDP were written in C# however, the OSCReceiver is written in JavaScript as it can identify the signal better than in C# as in C# it would not convert and object into an integer. Even though it is in JavaScript, the file had to be modified as it could not grab certain variables from a C# script, the bullet script had to be written within the OSCReceiver script. All assest where designed by myself, Adam Bader.

Processing

Leap Motion Sketch, which can be found in the Wekinator Examples. It was used for to send input positions for the game so for when the hand was at a “gun” position it would in act class 1 or if it were in a palm position it would call class 2 in Wekinator.

Wekinator

Using this software, it picked up what I was sending using the Leap Motion Sketch and would send out those features to the receiver script and act on it. For this, I used a classification k-nearest neighbour.

Algorithm

The algorithm that was used for this project was the k-nearest neighbour in which recognises the nearest neighbour on the position so if the “gun” position (class 1) was close the training set then it would go to class 1 and shoot, the reason for using this algorithm is because the leap motion can sometimes not recognise the hand position well enough so using that algorithm would be suitable as the aim is to make the user get as close as to the hand position as much as it possibly can.

Even though, there are other learning algorithm to use in a classification but reason a kNN is best to use as it does not need much training sets for this game so therefore there would not be much noise and would not be slow to recognise which class the user is on. While as the AdaBoost is usually fast to train and can deal with irrelevant features it would not be useful for a game that does not require much training sets for it.

Leap Motion

Used a leap motion to have hand detection so that players can use their hands to shoot the enemies that enter the level; this would give an immersive feel that they are actually using their hands to fire at enemies rather than using VR or Kinect.
